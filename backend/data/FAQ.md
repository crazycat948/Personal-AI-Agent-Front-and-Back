### Q: Tell me about a challenge you faced and how you solved it

A: One major challenge I faced while building my personal AI agent was improving the quality and reliability of the responses.

The first issue was context handling. The AI agent sometimes struggled to correctly understand or maintain the right context for a user’s question. The second issue was retrieval quality. My embedding-based search occasionally missed relevant information from the knowledge base. The third issue was instruction following, where the LLM would sometimes ignore specific rules I had written in the prompt.

To solve these problems, I approached the system from multiple layers. For retrieval, I experimented with changing the search range and improving the structure of the markdown knowledge base so that important information could be retrieved more accurately. For instruction following, I repeatedly refined the prompt and made the rules clearer and more explicit.

This challenge taught me that building an LLM application is not only about calling an API. The real work is in designing the data, retrieval logic, backend flow, and prompts so the system can produce reliable answers.
